# DigiUni Registry

## 1. Загальний опис

**DigiUni Registry** - консольна інформаційна система для обліку структури університету. Система дозволяє керувати:

- **Факультетами** - підрозділи університету верхнього рівня
- **Кафедрами** - структурні одиниці всередині факультетів
- **Студентами** - прив'язані до конкретної кафедри
- **Викладачами** - прив'язані до конкретної кафедри

Дані зберігаються у файлах CSV (папка `data/`) та можуть бути експортовані до JSON формату. Програма автоматично запускає HTTP-сервер на порту `8080` для моніторингу стану системи.

---

## 2. Вимоги до запуску
- Java 17 або вище
- Maven

## 3. Авторизація

При запуску система виводить тестові облікові дані:

```
admin admin
user user
manager manager
```

Якщо дані невірні - система запитає повторно.
### Вихід із системи

Оберіть пункт `0. Logout and exit` у головному меню. Це коректно завершить сесію та збереже дані.

---

## 4. Ролі та права доступу

У системі є три ролі:

**USER** - може лише переглядати дані та формувати звіти. Додавання, редагування, видалення та управління користувачами недоступні.

**MANAGER** - має всі права USER, а також може додавати та редагувати студентів, викладачів, факультети й кафедри. Видалення та управління користувачами недоступні.

**ADMIN** - повний доступ до всіх функцій системи, включно з видаленням записів та управлінням обліковими записами користувачів.

> ℹ️ Заблокований користувач (`blocked = true`) не може увійти в систему, навіть якщо пароль правильний.
---
## 5. Головне меню

Після успішної авторизації відкривається головне меню:

```
--- MAIN MENU ---
 0. Logout and exit
 1. Search and reports
--- MANAGER ONLY ---
 2. Add entity
 3. Update entity
--- ADMIN ONLY ---
 4. Delete entity
 5. Manage users
```

Введіть число відповідного пункту і натисніть `Enter`. Пункти, недоступні для вашої ролі, при виборі виводять повідомлення `Access denied`.

## 6. Пошук та звіти

> Доступно для всіх ролей.

Оберіть `1. Search and reports` у головному меню. Відкриється підменю:

```
--- SEARCH AND REPORTS ---
-- Search --
 1.  Find student by full name
 2.  Find student by grade
 3.  Find student by group
 4.  Find teacher by full name
-- Students --
 5.  All students sorted by grade
 6.  Students of faculty (alphabetically)
 7.  Students of department sorted by grade
 8.  Students of department (alphabetically)
 9.  Students of department by specific grade
-- Teachers --
10.  Teachers of faculty (alphabetically)
11.  Teachers of department (alphabetically)
-- Statistics --
12.  Student count by grade
13.  Student count by form of study
14.  Teacher count by department
15.  Average teacher rate by department
16.  Top departments by student count
 0.  Back
```

### Приклади використання

**Пошук студента за іменем (пункт 1):**
```
Enter full name (or part): Mokliak
```
Можна вводити частину імені - система знайде всіх відповідних студентів.

**Пошук студента за курсом (пункт 2):**
```
Enter grade (1-6): 3
```
Введіть номер курсу від 1 до 6.

**Пошук студента за групою (пункт 3):**
```
Enter group (1-3): 2
```
Введіть номер групи від 1 до 3.

**Пошук викладача за іменем (пункт 4):**
Аналогічно до пошуку студента - підтримується часткова відповідність.

**Студенти факультету (пункт 6):**
Система запропонує вибрати факультет зі списку - введіть його номер.

**Студенти кафедри за курсом (пункт 7) або за алфавітом (пункт 8):**
Система запропонує спочатку вибрати факультет, потім кафедру.

**Статистика:**
Пункти 12–16 виводять зведені дані без додаткових запитів.

> Підменю пошуку є циклічним - після кожного запиту воно повертається до списку. Щоб вийти, оберіть `0. Back`.

---

## 7. Додавання сутностей

> Доступно для ролей **MANAGER** та **ADMIN**.

Оберіть `2. Add entity`:

```
--- ADD ENTITY ---
1. Add Student
2. Add Faculty
3. Add Department
4. Add Teacher
0. Back
```

### 7.1 Додавання студента (пункт 1)

Система запитає:

1. **Кафедру** - вибір зі списку (спочатку факультет, потім кафедра)
2. **Прізвище** - тільки літери
3. **Ім'я** - тільки літери
4. **По батькові** - тільки літери
5. **Дату народження** - формат `YYYY-MM-DD`
6. **Email** - валідний формат електронної пошти
7. **Номер телефону** - рядок
8. **Курс** - число від 1 до 6
9. **Група** - число від 1 до 3
10. **Рік вступу** - число від 2000 до 2100
11. **Форма навчання** - `1 - Budget` (бюджет) або `2 - Contract` (контракт)
12. **Статус** - `1 - Studying` / `2 - Academic leave` / `3 - Deducted`

### 7.2 Додавання факультету (пункт 2)
Система запитає:
- Назву факультету
- Скорочену назву (абревіатуру)

### 7.3 Додавання кафедри (пункт 3)

Система запитає:
- Спочатку вибір факультету зі списку
- Назву кафедри
- Скорочену назву

### 7.4 Додавання викладача (пункт 4)

Система запитає:

1. **Кафедру** - вибір зі списку
2. **Прізвище, ім'я, по батькові** - тільки літери
3. **Дату народження** - формат `YYYY-MM-DD`
4. **Email** та **телефон**
5. **Посада** (Post) - наприклад, `Доцент`
6. **Науковий ступінь** (Degree) - наприклад, `Кандидат наук`
7. **Вчене звання** (Academic Rank) - наприклад, `Доцент`
8. **Дата початку роботи** - формат `YYYY-MM-DD`
9. **Рейтинг** (Rate) - число від 1 до 10

## 8. Редагування сутностей

> Доступно для ролей **MANAGER** та **ADMIN**.

Оберіть `3. Update entity`:

```
--- UPDATE ENTITY ---
1. Update Student
2. Update Faculty
3. Update Department
4. Update Teacher
0. Back
```

Для кожної сутності:
1. Система відобразить список наявних записів із номерами
2. Введіть номер запису для редагування
3. Система запропонує вибрати поле для зміни або оновити всі поля

> ℹ️ Якщо список порожній - спочатку додайте записи через меню `2. Add entity`.

---

## 9. Видалення сутностей

> Доступно **тільки для ADMIN**.

Оберіть `4. Delete entity`:

```
--- DELETE ENTITY ---
1. Delete Student
2. Delete Faculty
3. Delete Department
4. Delete Teacher
0. Back
```

Для кожної сутності:
1. Система відобразить список наявних записів
2. Введіть номер запису для видалення
3. Запис буде видалено негайно без підтвердження

> ⚠️ **Увага:** Видалення є незворотним у межах поточної сесії. Якщо ви не зробили збереження - можна перезавантажити дані з файлу через меню збереження.

---

## 10. Панель адміністратора

> Доступно **тільки для ADMIN**.

Оберіть `5. Manage users`:

```
--- ADMIN PANEL ---
-- Users --
1. View all users
2. Add user
3. Update user
4. Delete user
-- System --
5. Show all entities
6. Save / Load data (CSV)
7. Reflection and annotations
0. Back
```

### 10.1 Перегляд користувачів (пункт 1)

Виводить таблицю всіх облікових записів: логін, роль, статус блокування.

### 10.2 Додавання користувача (пункт 2)

Система запитає:
- **Логін** - унікальний рядок
- **Пароль**
- **Роль** - вибір: `USER`, `MANAGER`, або `ADMIN`

### 10.3 Редагування користувача (пункт 3)

Дозволяє змінити пароль, роль або статус блокування (`blocked`) існуючого облікового запису.

> ℹ️ Заблокований користувач не може увійти в систему. Цей спосіб зручний для тимчасового обмеження доступу без видалення акаунту.

### 10.4 Видалення користувача (пункт 4)

Видаляє обліковий запис. **Не можна видалити власний поточний акаунт.**

### 10.5 Показати всі сутності (пункт 5)

Відкриває підменю для перегляду повних списків:
- Всі студенти
- Всі факультети
- Всі кафедри
- Всі викладачі

## 11. Збереження та завантаження даних

Оберіть пункт `6. Save / Load data (CSV)` у панелі адміністратора:

```
--- SAVE / LOAD DATA ---
-- CSV (NIO.2) --
1. Save all data (CSV)
2. Load all data (CSV)
-- JSON (Jackson) --
3. Export to JSON
4. Import from JSON
0. Back
```

### 11.1 Збереження у CSV (пункт 1)

Записує поточний стан системи у файли:

| Файл | Вміст |
|------|-------|
| `data/university.csv` | Дані університету |
| `data/faculties.csv` | Список факультетів |
| `data/departments.csv` | Список кафедр |
| `data/students.csv` | Список студентів |
| `data/teachers.csv` | Список викладачів |
| `data/users.csv` | Облікові записи |


### 11.2 Завантаження з CSV (пункт 2)

Перезавантажує дані з файлів. **Поточні незбережені зміни будуть втрачені.**

### 11.3 Експорт у JSON (пункт 3)

Зберігає повний знімок бази даних у файл `data/backup.json`. Зручно для резервного копіювання та передачі даних.

### 11.4 Імпорт із JSON (пункт 4)

Завантажує дані з файлу `data/backup.json`. **Поточні дані в пам'яті будуть замінені.**

## 12. Відформатовані звіти

Оберіть пункт `7. Reflection and annotations` у панелі адміністратора:

```
--- REFLECTION / ANNOTATIONS ---
1. Show students details
2. Show teachers details
3. Show faculties details
4. Show departments details
5. Show class metadata
0. Back
```

Цей модуль відображає детальну внутрішню інформацію об'єктів за допомогою Java Reflection API:

- **Пункти 1–4** - виводять значення всіх полів відповідних сутностей у форматі `поле: значення`
- **Пункт 5** - виводить метадані класу: назву, поля, методи, конструктори

## 14. Логи

Програма веде детальний журнал подій у файлі `logs/digiUni.log`. Щодня створюється архівний файл з датою у назві.

У логах фіксуються:
- Входи та виходи користувачів (з тривалістю сесії)
- Спроби несанкціонованого доступу
- Операції збереження та завантаження даних
- Системні помилки

**Приклад запису в лозі:**
```
INFO  AuthService - LOGIN user=admin role=ADMIN
WARN  AuthService - ACCESS DENIED requires ADMIN, current=USER
INFO  AuthService - LOGOUT user=manager duration=5 min 23 sec
```

---

## 15. Типові помилки та їх вирішення

| Проблема | Причина | Рішення |
|----------|---------|---------|
| `Exiting to prevent data corruption` | Програма вже запущена | Закрийте інший екземпляр програми |
| `Access denied` при виборі пункту меню | Недостатньо прав | Увійдіть з акаунтом з відповідною роллю |
| Пустий список при спробі редагувати/видалити | Немає записів у системі | Спочатку додайте дані через `2. Add entity` |
| `Search query cannot be empty` | Введено порожній рядок пошуку | Введіть хоча б один символ |
| Некоректне введення числа | Введено текст замість числа | Введіть ціле число в зазначеному діапазоні |
| Дані не збережені після перезапуску | Не було виконано збереження | Перед виходом оберіть `Save all data (CSV)` |
| Файл `data/backup.json` відсутній | JSON-експорт не виконувався | Спочатку виконайте `Export to JSON` |
